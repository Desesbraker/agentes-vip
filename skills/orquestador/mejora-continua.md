# @mejora-continua

## OBJETIVO

Aplicar mejora continua sobre entregas y problemas recurrentes mediante cambios incrementales, prevencion de errores y estandarizacion. La skill convierte intuiciones de mejora en acciones con metodo.

## USAR CUANDO

- Ya hubo una entrega y se detectan fallos repetidos o deuda recurrente.
- Hace falta elegir entre 5 Whys, Fishbone, PDCA o A3.
- El equipo necesita evitar que el mismo error reaparezca.

## NO USAR CUANDO

- La tarea aun esta en fase de entendimiento; usar brainstorming.
- Se pretende justificar un rewrite grande sin evidencia.
- No existe problema observado ni patron repetido que analizar.

## INSTRUCCIONES

1. Describir el problema observado y elegir el metodo de analisis adecuado.
2. Identificar causa raiz, red flags y oportunidades de poka-yoke o trabajo estandarizado.
3. Definir cambios incrementales y medibles con validacion posterior.
4. Cerrar con aprendizaje capturado, accion correctiva y criterio de seguimiento.

## FORMATO DE SALIDA

- Problema y contexto.
- Metodo de analisis elegido.
- Causa raiz o causas principales.
- Cambios incrementales propuestos.
- Validacion y seguimiento.
