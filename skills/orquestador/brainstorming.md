# @brainstorming

## OBJETIVO

Descomponer solicitudes complejas o ambiguas hasta convertirlas en un entendimiento confirmado y opciones de enfoque comparables. La skill evita implementar antes de que el problema este realmente bloqueado.

## USAR CUANDO

- La solicitud es ambigua, compleja o mezcla varios objetivos.
- Hace falta entender restricciones no funcionales antes de planear.
- Se necesitan 2 o 3 enfoques para decidir direccion.

## NO USAR CUANDO

- La tarea ya esta clara y solo requiere plan corto; usar planificacion-concisa.
- El trabajo es ejecucion directa de un skill especializado.
- El usuario ya confirmo alcance, enfoque y restricciones relevantes.

## INSTRUCCIONES

1. Levantar contexto con una pregunta por vez y priorizar aclaraciones que cambian la solucion.
2. Confirmar requerimientos no funcionales y restricciones criticas.
3. Hacer understanding lock con resumen en bullets y confirmacion explicita antes de seguir.
4. Explorar 2 o 3 enfoques, registrar decision log y cerrar con opcion recomendada, riesgos y supuestos.

## FORMATO DE SALIDA

- Resumen confirmado del problema.
- Requerimientos no funcionales.
- Opciones evaluadas.
- Decision log.
- Riesgos, supuestos y siguiente paso.
