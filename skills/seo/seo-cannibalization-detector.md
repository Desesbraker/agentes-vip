# @seo-cannibalization-detector

OBJETIVO: Detectar y resolver conflictos entre URLs que compiten por la misma intención o keyword, evitando dilución de señales orgánicas.

USAR CUANDO: Hay sospecha de varias páginas targeting la misma búsqueda o cuando un cluster está creciendo y puede generar solapamientos.

NO USAR CUANDO: La tarea principal es crear nuevo contenido sin señales de conflicto entre URLs existentes. En ese caso basta con planning y keyword mapping.

INSTRUCCIONES:

1. Comparar keywords, intención, títulos, metas, headings y similitud temática entre URLs candidatas.
2. Distinguir si el conflicto es de intención, contenido, metadata o estructura para no aplicar la misma solución a todos los casos.
3. Resolver con consolidación, redirect, canonical, reescritura o rediseño hub-spoke según el conflicto real.
4. Cerrar con una prevención explícita: keyword mapping, revisión editorial y control pre-publicación.

FORMATO DE SALIDA:

- Keyword o intención en conflicto
- URLs afectadas
- Tipo de cannibalization
- Acción recomendada
- Riesgo de tráfico o indexación

RECURSOS RELACIONADOS:

- Ver `./_playbook.md` para checklist de contenido SEO.
- Ver `./_templates.md` para revisión de cannibalization.

ANTI-PATRÓN: Mantener varias páginas con el mismo target, consolidar sin redirect o canonical coherente, o ignorar la diferencia de intención entre URLs aparentemente similares.
