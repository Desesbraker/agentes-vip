# @seo-structure-architect

OBJETIVO: Diseñar arquitectura de contenido, headings y linking interno que facilite comprensión, indexación y cobertura temática sin crear caos estructural.

USAR CUANDO: Hay que definir o auditar la estructura de una página, sección o cluster desde la óptica de headings, silos e internal linking.

NO USAR CUANDO: El problema principal es calidad del copy, cannibalization entre URLs o schema específico. En esos casos usar la skill correspondiente.

INSTRUCCIONES:

1. Diseñar la jerarquía de headings alrededor de intención, tema y escaneabilidad, manteniendo una estructura semántica coherente.
2. Organizar silos y relaciones parent-child con enlaces internos contextuales, evitando cross-links arbitrarios que diluyan claridad temática.
3. Identificar oportunidades de snippet optimization y de schema complementario cuando la estructura del contenido lo justifique.
4. Cerrar con una recomendación estructurada que indique jerarquía, linking y riesgos de desorden o redundancia.

FORMATO DE SALIDA:

- Página o cluster
- Jerarquía de headings
- Silo o linking propuesto
- Oportunidad de snippet o schema
- Riesgo estructural

RECURSOS RELACIONADOS:

- Ver `./_playbook.md` para reglas duras del bloque.
- Ver `./_templates.md` para findings SEO.

ANTI-PATRÓN: Usar múltiples H1, romper la jerarquía semántica, crear silos con cross-links excesivos o optimizar snippets sin una estructura base clara.
