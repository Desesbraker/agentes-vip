# @programmatic-seo

OBJETIVO: Evaluar o diseñar una estrategia de SEO programático con criterios de viabilidad, valor único y sostenibilidad editorial.

USAR CUANDO: Hay que decidir si una iniciativa de SEO a escala merece inversión o cómo estructurarla sin crear thin content ni index bloat.

NO USAR CUANDO: Solo se necesita planificar unas pocas piezas manuales. En ese caso usar content planning normal.

INSTRUCCIONES:

1. Evaluar patrón de búsqueda, valor único por página, calidad de datos, intención, competencia y sostenibilidad antes de producir URLs a escala.
2. Elegir el playbook correcto según el tipo de página: comparativas, locations, glossaries, directories, integrations u otros formatos con valor real.
3. Definir quality gates y kill switch antes de lanzar, para evitar doorway pages, near-duplicates o páginas indexables sin propósito.
4. Cerrar con Feasibility Index, riesgos de supresión y recomendación clara de avanzar, pilotear o frenar.

FORMATO DE SALIDA:

- Feasibility Index
- Patrón programático propuesto
- Fuente de valor por página
- Riesgos de calidad o indexación
- Recomendación

RECURSOS RELACIONADOS:

- Ver `./_playbook.md` para reglas duras del bloque.
- Ver `./_templates.md` para findings SEO.

ANTI-PATRÓN: Crear doorway pages, llenar URLs con texto autogenerado sin valor, inflar el índice sin demanda real o usar datos públicos sin capa editorial diferenciadora.
