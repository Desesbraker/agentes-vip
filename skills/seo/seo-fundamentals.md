# @seo-fundamentals

OBJETIVO: Explicar o evaluar los principios base de SEO sin reducir la disciplina a hacks técnicos o listas de checks sin contexto.

USAR CUANDO: Hay que establecer criterios base de SEO, revisar fundamentos de un sitio o alinear al equipo sobre qué mueve impacto orgánico.

NO USAR CUANDO: La tarea ya exige una auditoría completa, un brief de contenido, schema o tracking específico. En esos casos usar la skill especializada.

INSTRUCCIONES:

1. Separar claramente fundamentos de contenido, autoridad, experiencia de página y baseline técnico para no sobredimensionar un solo frente.
2. Explicar E-E-A-T, Core Web Vitals, SEO técnico y contenido como partes de un sistema, no como checkboxes aislados.
3. Priorizar siempre la relación entre intención de búsqueda, valor del contenido y capacidad del sitio para ser rastreado e indexado correctamente.
4. Cerrar con una salida estructurada que deje claro qué base está sólida, qué está débil y qué no conviene sobreactuar.

FORMATO DE SALIDA:

- Pilar SEO evaluado
- Estado actual
- Impacto esperado
- Riesgo principal
- Siguiente prioridad

RECURSOS RELACIONADOS:

- Ver `./_playbook.md` para reglas duras del bloque SEO.
- Ver `./_templates.md` para findings SEO.

ANTI-PATRÓN: Optimizar solo técnico sin contenido útil, perseguir shortcuts temporales o invocar E-E-A-T como excusa vacía sin evidencia en el sitio o contenido.
