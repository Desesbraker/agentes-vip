# @analytics-product

OBJETIVO: Definir north star, funnels y métricas de éxito que conecten valor para el usuario con decisiones reales de producto.

USAR CUANDO: Hay que decidir qué medir para orientar roadmap, evaluar iniciativas o alinear producto con datos y negocio.

NO USAR CUANDO: El tracking base aún no existe o los eventos clave no están definidos. Coordinar primero con Data Analyst y equipos de implementación.

INSTRUCCIONES:

1. Elegir una north star conectada al valor para el usuario y al resultado de negocio, evitando métricas que solo reflejen actividad interna.
2. Diseñar funnel y métricas proxy por etapa con evento fuente, fórmula y dueño claros para cada indicador relevante.
3. Separar leading y lagging indicators para no esperar demasiado a confirmar si una iniciativa funciona ni sobre-reaccionar a ruido temprano.
4. Cerrar con umbrales de éxito, frecuencia de revisión y decisiones asociadas para que cada métrica tenga una consecuencia operativa.

FORMATO DE SALIDA:

- North star metric
- Funnel y etapas medidas
- Eventos, fórmulas y owners
- Umbrales y frecuencia de revisión
- Riesgos de tracking o métricas vanity

RECURSOS RELACIONADOS:

- Ver `./_playbook.md` para checklist de analytics de producto.
- Ver `./_templates.md` para la product metrics sheet.

ANTI-PATRÓN: Tener muchas métricas principales compitiendo, medir vanity metrics o elegir una north star que no cambie ninguna decisión de producto.
