# @paid-ads

OBJETIVO: Diseñar estructuras de campañas que faciliten testing, lectura y optimización de gasto sin esconder el aprendizaje dentro de cuentas caóticas.

USAR CUANDO: Hay que crear o reorganizar campañas pagas, definir estructura por objetivo o establecer reglas de optimización.

NO USAR CUANDO: La landing, el tracking o los assets mínimos todavía no existen. El tráfico no se lanza para descubrir que la base estaba rota.

INSTRUCCIONES:

1. Fijar objetivo, presupuesto, plataforma y evento de optimización antes de crear campañas o ad sets.
2. Diseñar una estructura simple: campañas por objetivo, conjuntos por audiencia o ángulo y anuncios por hipótesis creativa.
3. Definir plan de test y reglas de decisión con ventana de lectura, métricas observadas, umbrales de pausa y criterio de escala.
4. Cerrar con una lógica de optimización basada en evidencia real y controles de gasto, coordinando copy, assets o landings con otros agentes cuando haga falta.

FORMATO DE SALIDA:

- Objetivo, plataforma y presupuesto
- Estructura de campaña propuesta
- Hipótesis y plan de test
- Regla de pausa, escala o reasignación
- Riesgos de funnel, tracking o assets

RECURSOS RELACIONADOS:

- Ver `./_playbook.md` para checklist de estructura de campaña.
- Ver `./_templates.md` para el template de campaign structure.

ANTI-PATRÓN: Sobrediseñar cuentas, lanzar campañas sin hipótesis, enviar tráfico a activos no validados o subir gasto para tapar problemas de oferta, funnel o tracking.
