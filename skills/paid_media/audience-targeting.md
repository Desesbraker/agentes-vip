# @audience-targeting

OBJETIVO: Definir audiencias y segmentos que mejoren relevancia, escalabilidad y lectura de resultados sin contaminar experimentos entre capas del funnel.

USAR CUANDO: Hay que diseñar prospecting, retargeting, expansión o lookalikes para una campaña paga.

NO USAR CUANDO: No existe ICP, intención o señal base confiable. La plataforma no corrige una estrategia de segmentación mal definida.

INSTRUCCIONES:

1. Partir de ICP, intención, exclusiones y etapa del funnel antes de crear audiencias en plataforma.
2. Separar prospecting, retargeting y expansión para que cada capa tenga mensaje, presupuesto y KPI acordes a su objetivo.
3. Construir audiencias con señales confiables como first-party data, comportamiento onsite, listas limpias o lookalikes con seed de calidad.
4. Cerrar con revisión de solapamiento, saturación, escala y sesgo de segmentación para evitar canibalización entre experimentos.

FORMATO DE SALIDA:

- Segmentos y objetivo por audiencia
- Tipo de audiencia y fuente
- Exclusiones y solapamientos
- KPI por capa
- Riesgos de escala, canibalización o mala señal

RECURSOS RELACIONADOS:

- Ver `./_playbook.md` para checklist de audiencias.
- Ver `./_templates.md` para la audience matrix.

ANTI-PATRÓN: Crear audiencias enormes sin intención, construir lookalikes sobre seeds pobres o mezclar retargeting con prospecting dentro del mismo experimento.
