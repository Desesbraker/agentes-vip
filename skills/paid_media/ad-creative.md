# @ad-creative

OBJETIVO: Crear y testear creatividades publicitarias que permitan aprender qué hook, ángulo y formato mueven mejor el objetivo de campaña.

USAR CUANDO: Hay que producir variantes de anuncio, definir hipótesis creativas o organizar un test de assets por plataforma.

NO USAR CUANDO: No existe hipótesis, audiencia o oferta definidas. En ese caso todavía no hay test; solo producción estética sin lectura causal.

INSTRUCCIONES:

1. Definir la hipótesis creativa con hook, ángulo, oferta, formato y audiencia antes de producir cualquier pieza.
2. Adaptar cada asset a plataforma y placement con ratio, duración, primer frame, subtítulos, branding y CTA coherentes con el consumo real del anuncio.
3. Testear pocas variables por vez para preservar causalidad: hook, visual, prueba social, oferta o CTA, pero no todo a la vez.
4. Cerrar con una lectura de fatigue y winners por cohorte o segmento, decidiendo si conviene iterar el asset, pedir material nuevo o pausar.

FORMATO DE SALIDA:

- Hipótesis creativa
- Variantes y variable de test
- Formato por placement
- Criterio para winner o fatigue
- Riesgos de saturación o falta de hook

RECURSOS RELACIONADOS:

- Ver `./_playbook.md` para checklist de creatividades.
- Ver `./_templates.md` para la creative test matrix.

ANTI-PATRÓN: Crear piezas bonitas sin hook, publicar anuncios sin subtítulos o mezclar tantas variables por versión que luego no se pueda aprender nada útil del test.
