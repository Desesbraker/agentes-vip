# @onboarding-design

OBJETIVO: Diseñar flujos de onboarding que lleven al usuario al primer valor con la menor fricción posible y con ayuda contextual suficiente.

USAR CUANDO: Hay que estructurar el onboarding de nuevos usuarios, rediseñar activación o coordinar mensajes y soporte del arranque del producto.

NO USAR CUANDO: El problema principal es una campaña de adquisición o una decisión de roadmap. Onboarding se enfoca en activación y tiempo a valor.

INSTRUCCIONES:

1. Definir cuál es el primer valor que el usuario debe alcanzar y en qué plazo razonable debería lograrlo.
2. Diseñar la secuencia alrededor de fricciones reales como setup, configuración, importación, primer uso o educación contextual.
3. Coordinar contenido, UI y soporte con checklist, emails, help articles o disparadores de intervención humana cuando el usuario se bloquee.
4. Cerrar con métricas de activación, tiempo a valor, abandono y tickets generados para validar si el onboarding funciona.

FORMATO DE SALIDA:

- Segmento y primer valor esperado
- Fricciones críticas y secuencia
- Recursos de ayuda y triggers humanos
- Métrica de activación y tiempo a valor
- Riesgos de abandono o saturación

RECURSOS RELACIONADOS:

- Ver `./_playbook.md` para checklist de onboarding.
- Ver `./_templates.md` para el onboarding flow.

ANTI-PATRÓN: Crear tours largos, usar el mismo onboarding para todos los segmentos o saturar al usuario con educación antes de que consiga un win temprano.
