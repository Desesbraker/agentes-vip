# @interactive-portfolio

## OBJETIVO

Diseñar portfolios interactivos que comuniquen identidad, credibilidad y calidad de trabajo en pocos segundos.

## USAR CUANDO

Hay que estructurar un portfolio personal o profesional para un diseñador, developer o perfil híbrido.

## NO USAR CUANDO

La necesidad es un sitio corporativo, un CV plano o una landing de producto sin narrativa personal.

## INSTRUCCIONES

1. Validar el 30-Second Test: en medio minuto debe quedar claro quién es la persona, qué hace, cuál es su mejor trabajo y cómo contactarla.
2. Ordenar secciones con un objetivo claro de conversión y prueba: hero, trabajo, caso de estudio, sobre mí y contacto.
3. Diseñar cada case study como evidencia, no como galería: reto, rol, decisiones, proceso, resultado y links útiles.
4. Cerrar con una dirección visual propia y una navegación que favorezca exploración sin esconder lo importante.

## FORMATO DE SALIDA

- Identidad y objetivo del portfolio
- Secciones y narrativa
- Caso de estudio modelo
- Elementos de credibilidad
- Riesgo de parecer plantilla

## RECURSOS RELACIONADOS

- Ver `./_playbook.md` para checklist de entrega y calidad visual.
- Ver `./_templates.md` para el template de dirección visual o design system.
- Ver `./resources/portfolio-playbook.md` para 30-Second Test, estructura, case studies y señales de credibilidad.

## ANTI-PATRON

Portfolio genérico, mucho estilo y poca prueba, sitio que parece CV sin personalidad o navegación que oculta el trabajo real.
