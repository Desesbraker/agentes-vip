# @frontend-design

## OBJETIVO

Definir una dirección estética memorable para interfaces digitales, conectando intención visual con decisiones implementables.

## USAR CUANDO

Hay que fijar el look and feel de una interfaz, explorar una propuesta visual o evitar que el frontend caiga en un patrón genérico.

## NO USAR CUANDO

Ya existe un design system o una marca cerrada que solo hay que aplicar con disciplina. En ese caso prima consistencia sobre exploración.

## INSTRUCCIONES

1. Aclarar el tono, la audiencia y el differentiation anchor de la interfaz antes de hablar de componentes o librerías.
2. Evaluar factibilidad e impacto visual con una heurística explícita, como DFII, para evitar direcciones bellas pero inviables.
3. Traducir la dirección en reglas concretas de tipografía, color, composición y motion que luego puedan implementar Frontend o Mobile.
4. Cerrar con una síntesis visual accionable, dejando claro qué hace única a la interfaz y qué riesgos amenazan esa identidad.

## FORMATO DE SALIDA

- Objetivo visual
- Dirección estética elegida
- Reglas tipográficas y cromáticas
- Composición y motion
- Riesgo de genericidad o inviabilidad

## ANTI-PATRON

Layouts genéricos, gradientes SaaS intercambiables, tipografía por defecto o decoración sin intención defendible.
