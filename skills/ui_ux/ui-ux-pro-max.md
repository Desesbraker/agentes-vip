# @ui-ux-pro-max

## OBJETIVO

Diseñar sistemas UI/UX consistentes, accesibles y listos para implementación, priorizando claridad de uso sobre ornamento.

## USAR CUANDO

Hay que crear componentes, revisar UX, definir un design system o evaluar la calidad visual e interactiva de una interfaz.

## NO USAR CUANDO

La tarea solo exige copy, lógica de negocio o deploy técnico. Esta skill resuelve experiencia e interfaz.

## INSTRUCCIONES

1. Priorizar accesibilidad, interacción y performance antes de estilo, asegurando que cualquier decisión visual soporte uso real.
2. Definir o auditar el design system con tipografía, color, spacing, componentes, estados y anti-patrones explícitos.
3. Validar interacción con contraste suficiente, focus visible, navegación por teclado, targets táctiles y feedback consistente.
4. Entregar decisiones listas para implementación, incluyendo riesgos, vacíos y reglas que el equipo técnico debe preservar.

## FORMATO DE SALIDA

- Objetivo de experiencia
- Sistema visual y componentes
- Reglas de accesibilidad e interacción
- Riesgos o vacíos detectados
- Siguiente paso de implementación

## ANTI-PATRON

Emojis como iconos, hover con layout shift, contraste insuficiente, ausencia de focus states o UI vistosa que no resiste uso real.
