# Generative and Canvas Visual Playbook

## Cuándo abrir este recurso

- Diseñar piezas canvas o editoriales
- Crear arte generativo con p5.js u otra lógica reproducible
- Resolver si una pieza debe ser sistema o composición fija

## Elegir enfoque

| Tipo | Usar cuando |
|---|---|
| Canvas design | la pieza es fija, editorial, poster o composición controlada |
| Algorithmic art | el sistema generativo es parte del valor artístico |

## Flujo operativo

1. Definir concepto y filosofía visual.
2. Decidir si la obra vive como composición fija o como sistema generativo.
3. Fijar paleta, restricciones, reglas y límites.
4. Validar export, reproducibilidad y legibilidad del resultado.

## Para algorithmic art

- Definir seed reproducible
- Definir variaciones paramétricas
- Limitar el azar con reglas visuales claras
- Incluir controles mínimos si hay exploración

## Para canvas design

- Definir grilla o lógica compositiva
- Evitar superposición accidental
- Priorizar jerarquía y balance
- Diseñar para output claro en PNG, PDF o formato previsto

## Checklist

- [ ] Concepto claro
- [ ] Sistema o composición correctamente elegido
- [ ] Seed o restricciones definidas
- [ ] Output reproducible o controlado
- [ ] La interfaz no eclipsa la pieza

## Entrega esperada

- Concepto
- Sistema visual o generativo
- Restricciones formales
- Entregables
- Riesgo de ruido o pérdida de identidad
