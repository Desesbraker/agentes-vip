# Portfolio Playbook

## Cuándo abrir este recurso

- Diseñar portfolio para developer, designer o perfil híbrido
- Necesitar narrativa, prueba de trabajo y dirección visual propia

## Regla base

El portfolio no es un CV largo ni una galería vacía. En 30 segundos debe responder:

- quién es la persona
- qué hace
- cuál es su mejor trabajo
- por qué confiar en ella
- cómo contactarla

## Estructura sugerida

1. Hero claro
2. Trabajo destacado
3. Casos de estudio
4. Sobre mí
5. Contacto

## Case study model

Cada caso debería mostrar:

- reto
- rol
- decisiones
- proceso
- resultado
- links o artefactos útiles

## Credibilidad

- métricas o impacto si existen
- logos o contexto real sin exagerar
- testimonios solo si son reales
- links a producto, repo o walkthrough si aportan confianza

## Checklist

- [ ] 30-Second Test pasa
- [ ] El trabajo real se encuentra rápido
- [ ] Cada caso prueba criterio y no solo estética
- [ ] La navegación no oculta lo importante
- [ ] No parece template genérico

## Entrega esperada

- Identidad y objetivo del portfolio
- Orden de secciones
- Modelo de case study
- Mecanismos de credibilidad
- Riesgo de parecer plantilla o CV disfrazado
