Guía operativa extendida para la familia UI/UX.

La skill principal sigue siendo corta y activable. Estos recursos recuperan profundidad útil del upstream sin inflar el perfil ni convertir cada skill en un bloque monolítico.

## Mapa de uso

- `design-system-playbook.md`
  - Usar con `@ui-ux-pro-max`
- `frontend-aesthetics-playbook.md`
  - Usar con `@frontend-design`
- `mobile-ux-playbook.md`
  - Usar con `@mobile-design`
- `portfolio-playbook.md`
  - Usar con `@interactive-portfolio`
- `generative-visual-playbook.md`
  - Usar con `@algorithmic-art` y `@canvas-design`

## Regla operativa

1. La skill corta define trigger, salida y guardrails.
2. El playbook correspondiente aporta workflow, checklist, criterios de calidad y referencias de implementación.
3. Si el problema requiere decisiones más finas de plataforma o exploración, abrir el resource antes de diseñar.
