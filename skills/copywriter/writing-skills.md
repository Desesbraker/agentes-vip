# @writing-skills

OBJETIVO: Crear o mejorar skills y playbooks para agentes de la agencia con triggers claros, límites explícitos y estructura que otros agentes puedan ejecutar sin racionalizar reglas.

USAR CUANDO: La tarea consiste en diseñar, reescribir o endurecer una skill, playbook o recurso operativo para agentes internos.

NO USAR CUANDO: El trabajo es redactar copy comercial, contenido de marca o documentación de producto para usuarios finales. En esos casos usar la skill de escritura correspondiente.

INSTRUCCIONES:

1. Definir el tier de la skill: Tier 1 simple para piezas cortas, Tier 2 expanded para workflows medianos y Tier 3 platform para sistemas con varios productos o fronteras. El tier marca el nivel de detalle esperado.
2. Aplicar CSO, Content Search Optimization: descripción breve con triggers de uso claros y keywords suficientes para que el agente encuentre la skill cuando la necesite.
3. Escribir reglas anti-rationalization, es decir, instrucciones difíciles de reinterpretar o ignorar. Cuando una regla sea crítica, expresarla como obligación o bloqueo.
4. Validar estructura, naming y pruebas de uso real. RED-GREEN-REFACTOR significa definir escenario, producir una versión mínima que funcione y después refinar sin perder activación, claridad ni seguridad.

FORMATO DE SALIDA:

- Objetivo y trigger de la skill
- Tier y alcance
- Estructura propuesta o cambios
- Reglas críticas y anti-rationalization
- Escenarios de prueba y riesgos

RECURSOS RELACIONADOS:

- Ver `./_playbook.md` para glosario de CSO, tiers y RED-GREEN-REFACTOR.
- Ver `./_templates.md` para el brief de copy y el registro de edición cuando toque reescritura parcial.

ANTI-PATRÓN: Descripciones que narran workflow en vez de trigger, nombres genéricos, skills monolíticas sin anexos o reglas críticas redactadas como sugerencias blandas.
