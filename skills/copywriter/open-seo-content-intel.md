# @open-seo-content-intel

OBJETIVO: Convertir datos SEO verificables en un brief editorial útil para redactar contenido orgánico sin inventar métricas ni sacrificar la voz de marca.

USAR CUANDO: Hace falta research SEO real para blogs, landings orgánicas, content refresh, metadatos o topic clusters antes de escribir.

NO USAR CUANDO: Ya existe research SEO validado y la tarea principal es redactar la pieza final. En ese caso usar @content-creator o @copywriting según el formato.

INSTRUCCIONES:

1. Consultar el playbook compartido de OpenSEO y usar la herramienta local como fuente verificable de inputs. Extraer solo datos que puedan atribuirse a una consulta o workflow concreto.
2. Identificar keyword principal, variantes, intención de búsqueda, preguntas relacionadas y oportunidades de internal linking. Intención puede ser informational, commercial, navigational o transactional.
3. Convertir esos datos en un brief editorial con ángulo, estructura, títulos candidatos, meta description y recomendaciones de enlaces internos.
4. Si OpenSEO no está disponible o falta la API key requerida, reportar bloqueo parcial, pedir apoyo al SEO Specialist vía Orquestador o declarar placeholders explícitos sin inventar volumen ni dificultad.

FORMATO DE SALIDA:

- Keyword principal e intención
- Variantes y preguntas a cubrir
- Internal links sugeridos
- Brief editorial y draft angle
- Limitaciones, placeholders o bloqueos operativos

RECURSOS RELACIONADOS:

- Ver `./_playbook.md` para glosario editorial y checklist de contenido.
- Ver `./_templates.md` para el outline de contenido.
- Ver `../shared/open-seo-playbook.md` para el flujo operativo y plantillas mínimas de OpenSEO.

ANTI-PATRÓN: Hacer keyword stuffing, inventar volumen o dificultad, tratar OpenSEO como sustituto de la voz de marca o ocultar que la research quedó bloqueada por falta de datos o de API key.
