# @copy-editing

OBJETIVO: Mejorar claridad, tono, prueba y fricción de un texto existente sin destruir su mensaje central ni la voz del autor.

USAR CUANDO: Existe un draft o pieza ya escrita que necesita revisión, pulido o una pasada estructurada de mejora antes de publicarse.

NO USAR CUANDO: El problema real es que todavía no existe un brief, una estrategia o una estructura base. En ese caso usar primero @copywriting, @content-creator o @doc-coauthoring según la pieza.

INSTRUCCIONES:

1. Aplicar los Seven Sweeps en orden: Clarity, Voice/Tone, So What, Prove It, Specificity, Heightened Emotion y Zero Risk. Ver definiciones operativas en el playbook.
2. Después de cada sweep, hacer loop-back para verificar que la mejora no rompió claridad, tono o intención de una pasada anterior.
3. Hacer un quick pass para cortar filler words, reemplazar corporate speak, eliminar voz pasiva innecesaria y simplificar frases que cargan más de una idea.
4. Entregar una edición justificada, preservando la voz del autor y señalando dónde faltan pruebas, datos o decisiones que el editor no debe inventar.

FORMATO DE SALIDA:

- Objetivo de la edición
- Problemas detectados por sweep
- Cambios clave realizados
- Claims o datos que requieren validación
- Riesgos de voz, claridad o fricción remanentes

RECURSOS RELACIONADOS:

- Ver `./_playbook.md` para el detalle de Seven Sweeps y el checklist de edición.
- Ver `./_templates.md` para el registro de edición.

ANTI-PATRÓN: Reescribir el mensaje core sin permiso, editar todo en una sola pasada, justificar cambios con gusto personal o corregir claims inventando evidencia que no existe.
