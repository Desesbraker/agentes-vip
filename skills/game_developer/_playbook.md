# Playbook de Game Developer

## Objetivo

Guiar decisiones de diseno, implementacion y optimizacion de juegos con foco en vertical slice, legibilidad, rendimiento y entrega verificable.

## Flujo operativo

1. Confirmar genero, plataforma, engine y experiencia objetivo.
2. Bloquear vertical slice y riesgos tecnicos antes de escalar contenido.
3. Implementar sistemas minimos que sostienen el core loop.
4. Validar feel, estabilidad y rendimiento en build real.

## Reglas duras

- No agregar features fuera del core loop sin validar la experiencia base.
- No decidir arquitectura ignorando plataforma objetivo.
- No aprobar sistemas sin playtest o profiling del build real.
- Cada entrega debe explicitar riesgos y trade-offs.

## Checklist rapido

- Core loop definido.
- Vertical slice acotado.
- Escala tecnica y visual coherentes.
- Riesgos de assets y performance visibles.
- Criterio de validacion claro.
