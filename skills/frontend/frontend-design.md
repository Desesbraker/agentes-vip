# @frontend-design

## OBJETIVO

Definir una dirección visual intencional para interfaces frontend que se vea específica, coherente y alineada con el propósito del producto.

## USAR CUANDO

Hay que decidir look and feel, sistema visual base, tipografía, color, composición o una dirección estética antes de implementar UI.

## NO USAR CUANDO

La tarea ya viene cerrada por una marca o design system existente; en ese caso se debe preservar el lenguaje visual definido.

## INSTRUCCIONES

1. Fijar propósito, tono estético y un differentiation anchor claro antes de elegir componentes o efectos visuales.
2. Limitar la dirección a pocas decisiones fuertes: una línea tipográfica principal, una secundaria y una paleta corta con roles explícitos.
3. Traducir esa dirección a reglas aplicables: densidad, contraste, radios, sombras, ritmo espacial y tipo de motion permitido.
4. Entregar una síntesis accionable que conecte intención visual con implementación y evite caer en layouts genéricos o estilos intercambiables.

## FORMATO DE SALIDA

- Objetivo visual
- Dirección estética elegida
- Sistema tipográfico y color
- Reglas de composición o motion
- Riesgo de caer en genérico

## RECURSOS RELACIONADOS

- Ver `./_playbook.md` para reglas duras de dirección visual y validación.
- Ver `./_templates.md` para el template de dirección visual.

## ANTI-PATRON

Usar fuentes genéricas por defecto, caer en gradientes SaaS intercambiables, copiar layouts de plantilla o decorar sin una dirección estética defendible.
