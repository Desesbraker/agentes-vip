# Tailwind and Frontend Design System Playbook

## Cuándo abrir este recurso

- Implementar diseño con Tailwind de forma consistente
- Resolver tokens, variantes y responsive por componente
- Aterrizar dirección visual a utilidades reutilizables

## Flujo operativo

1. Bloquear tokens y roles visuales.
2. Decidir si la responsividad depende de viewport o componente.
3. Extraer patrones repetidos a componentes o variantes.
4. Validar contraste, spacing y legibilidad antes de cerrar.

## Capas de token

- primitivo
- semántico
- componente

## Reglas útiles

- Preferir variables y theme antes que arbitrary values indiscriminados.
- Usar container queries cuando el componente lo requiera.
- No usar Tailwind como sustituto de ausencia de sistema.

## Checklist

- [ ] Tokens claros
- [ ] Responsive strategy definida
- [ ] Reutilización prevista
- [ ] Riesgo de inconsistencia controlado
- [ ] Contraste y legibilidad validados

## Entrega esperada

- Patrón visual
- Tokens implicados
- Estrategia responsive
- Reutilización prevista
- Riesgo de inconsistencia
